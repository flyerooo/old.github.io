        • localStorage 与 sessionStorage 区别
          • 通过localStorage所存储的数据，除非开发者主动把它删除或是用户通过浏览器选项对其清理，否则就会一直保留下去，实际上用户几乎不会去清理这些数据
          • sessionStorage则不同，由于它是一种基于会话（session-base）的存储方式，所以每次浏览器会话结束（浏览器被关闭）时，其中的数据就会被自动移除

    • 第七章 以事件处理实现用户交互
      • 事件处理模型
        • 获取DOM节点
        • 在其中设置事件处理程序
        • 如果事件被触发，那么就执行用于处理该事件的特定函数
      • 设置事件处理程序
        • event handler：事件处理器
          • 找到那个元素
          • 通过on属性在钙元素中设置发生事件时所要执行的代码
        • event listener：事件监听器（更好的方法）
          • 找到需要监听的DOM元素
          • 确定需要监听的事件类型
          • 传入事件监听函数
          • 传入一个表示是否启用“捕获模式”的Boolean值
          • element.addEventListener(
          • 兼容性
            • IE9及更高版本
              • IE9已下可用attachEvent（），区别它不需要那个代表是否启用捕获模式的Boolean值。
            • var btn = document.getElementById(‘btn’); if (btn.addEventListener){ btn.addEventListener(‘click’,function () { alert(‘clicked the button’); }, false); } else { element.attachEvent(‘click’,function () { alert(‘clicked the button’); }); }
        • 事件绑定
          • 调用addEventListener()时，与其它代码不同，后面是不带那一对括号的
          • 如果要在调用addEventListener()方法时向函数传入参数，那么你得把那个函数套在一个匿名函数里
            • var btn = document.getElementById(‘btn’); function alertMessage(message) { alert(message); } btn.addEventListener(‘click’,function () { alertMessage(‘clicked the button’); }, false);
        • 解除事件绑定
          • 大部分时间添加完事件处理程序就不用管它了，如果想解除可以使用
            • if(btn.removeEventListener){ btn.removeEventListener(‘click’, alertMessage, false); } else { btn.detachEvent(‘click’,alertMessage); }
